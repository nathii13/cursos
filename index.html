Comandos

Git Clone + enlace ssh de GIT

Replicamos el repositorio creado en GIT, ahora en una carpeta local

Git branch + nombre de rama

creamos nuevas ramas dentro del proyecto

Git switch + nombre de rama

nos permite cambiar entre una rama y otra

Git add .

agrega TODOS los archivos a staging

Git status

Nos indica el estado de los archivos: si están en staging, sin cambios, limpios o en repositorio para producciòn

Git push

sube los cambios al repositorio de GIT. Va despues del commit

Git commit -m

pasa los cambios de staging a una instacia de pre produccion

Git pull origin + nombre rama

Trae los cambios de la rama indicada

Git checkout

Cambiar entre diferentes versiones de una entidad de destino.

Git diff + nombre del archivo (index.html)

Muestra las diferencias que tienen el archivo, es decir los cambios guardados en el archivo pero sin add al stage.

Git commit 2.0

Si se da enter sin -m, se puede dar i, se redacta el texto y luego se presiona esc + :wq para volver a la terminal

.gitignore

Se puede crear un archivo en el vscode que se llama .gitignore, donde se colocan archivos y carpetas que se desea que no se guarden ni se copien en el repositorio.

git branch

Muestra las ramas que tiene el proyecto: main / * practica_git_1. Cuando aparece el * eso significa la rama donde se está trabajando

como navegar entre carpetas con GIT

ls

muestra el contenido de la carpeta local actual

cd + nombre carpeta

permite acceder a través de la terminal a la carpeta indicada

cd ..

permite regresar a la carpeta contenedora de la carpeta inicial. Permite devolverse un nivel
